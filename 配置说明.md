# Presentation AI - 环境配置说明

## ✅ 已完成的配置

### Google OAuth (已配置)
- ✅ GOOGLE_CLIENT_ID: 已从您的客户端密钥文件提取
- ✅ GOOGLE_CLIENT_SECRET: 已从您的客户端密钥文件提取
- ✅ NEXTAUTH_SECRET: 已生成安全密钥
- ✅ NEXTAUTH_URL: 已设置为本地开发地址

## 🔧 需要您手动配置的项目

### 1. 数据库配置 (必需)
```env
DATABASE_URL="postgresql://用户名:密码@主机:端口/数据库名"
```

**PostgreSQL 设置步骤:**
1. 确保已安装 PostgreSQL
2. 创建数据库: `CREATE DATABASE presentation_ai;`
3. 更新 `.env.local` 中的连接字符串

### 2. AI 服务 API 密钥 (必需)

#### OpenAI API (内容生成)
- 访问: https://platform.openai.com/api-keys
- 创建新的API密钥
- 更新 `OPENAI_API_KEY`

#### Together AI (图像生成)
- 访问: https://together.ai/settings/api-keys
- 注册并创建API密钥
- 更新 `TOGETHER_AI_API_KEY`

### 3. 其他服务 (可选)

#### UploadThing (文件上传)
- 访问: https://uploadthing.com/
- 注册应用并获取Token
- 更新 `UPLOADTHING_TOKEN`

#### Unsplash (图片搜索)
- 访问: https://unsplash.com/developers
- 创建应用获取Access Key
- 更新 `UNSPLASH_ACCESS_KEY`

#### Tavily AI (网络搜索)
- 访问: https://tavily.com/
- 注册并获取API密钥
- 更新 `TAVILY_API_KEY`

## 🚀 启动项目

1. **安装依赖**
```bash
pnpm install
```

2. **数据库初始化**
```bash
pnpm db:push
```

3. **启动开发服务器**
```bash
pnpm dev
```

4. **测试登录**
- 访问: http://localhost:3000
- 点击登录，使用Google账号认证

## 📁 文件说明

- `.env.local`: 本地环境变量配置 (不会被git跟踪)
- `client_secret_...json`: Google OAuth凭据文件 (不要提交到版本控制)

## 🔍 故障排除

### 1. Google登录失败
- 检查客户端ID和Secret是否正确
- 确保 `NEXTAUTH_URL` 与Google Console中的重定向URI匹配

### 2. 数据库连接失败
- 验证PostgreSQL服务是否运行
- 检查数据库连接字符串格式
- 确保数据库已创建

### 3. API密钥错误
- 确保所有API密钥都已正确配置
- 检查密钥是否有效且未过期

## 📝 快速配置模板

复制以下模板到 `.env.local` 并替换为您实际的密钥：

```env
# Google OAuth (需要配置)
GOOGLE_CLIENT_ID="您的Google OAuth Client ID"
GOOGLE_CLIENT_SECRET="您的Google OAuth Client Secret"

# NextAuth (需要配置)
NEXTAUTH_URL="http://localhost:3000"
NEXTAUTH_SECRET="您的NextAuth Secret密钥"

# 数据库 - 需要您配置
DATABASE_URL="postgresql://您的用户名:您的密码@localhost:5432/presentation_ai"

# AI服务 - 需要您配置
OPENAI_API_KEY="您的OpenAI密钥"
TOGETHER_AI_API_KEY="您的Together AI密钥"

# 其他服务 - 可选配置
UPLOADTHING_TOKEN="您的UploadThing Token"
UNSPLASH_ACCESS_KEY="您的Unsplash密钥"
TAVILY_API_KEY="您的Tavily密钥"

# 开发环境
NODE_ENV="development"
```
